Guide

Install

npm install -g dotdog        # prerequisite
cd dogbench && pip install -e .

dotdog is the spec engine that powers .dog → .dag compilation. Installed as npm dep.

Run

./dogbench                    # collars vs hermes vs raw API
./dogbench --all              # all detected agents
./dogbench --submit           # auto-PR to leaderboard
./dogbench "your task here"   # custom benchmark task

Task Design

The benchmark task must require project understanding — otherwise DAG injection provides no advantage. The default task asks "what agents does dogbench support, what's the benchmark flow?" — Collar answers from injected DAG; Hermes/claude/codex must read files.

Three Tiers

TierAgentWhat it measures
1DeepSeek API (raw)Theoretical minimum — no tools, no file access, just the model. Shown as ⚡ estimate when no API key.
2hermes (upstream)Standard harness without DAG injection. Reads AGENTS.md, explores files.
3collar + dotdog (DAG-first)Same harness with .dag pre-loaded. Skips exploration — proves DAG saves tokens.

How It Works

Dogbench runs an identical task through each agent's CLI. It reads official usage logs — collar/hermes from ~/.dag/state.db (or ~/.hermes/state.db), Claude from ~/Library/Application Support/Claude/usage/, Codex from ~/.codex/usage/. No estimates, no bias. Your own hardware, your own tokens.

Submit Results

./dogbench --submit auto-creates a PR. CI validates the schema and auto-merges. Results appear on this leaderboard within minutes.

Specs

Dogbench is specdog-compliant. Specs live in specs/dogbench/ — SPEC.dog, constitution.dog, data-model.dog. Compile with npx dotdog compile.
